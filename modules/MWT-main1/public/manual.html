Manuale d'Uso - Masters Winter Tour Calculator

Questo manuale descrive il funzionamento del sito del Masters Winter Tour, con particolare attenzione alle funzionalità di calcolo delle classifiche e al controllo degli accessi.

1. Panoramica del Sito

Il sito è una Single Page Application (SPA) distribuita su Cloudflare Pages, con funzionalità di backend implementate come Cloudflare Functions. Utilizza un database D1 per la persistenza dei dati e un sistema di autenticazione basato su JWT per proteggere le funzionalità amministrative.

Struttura delle Pagine:

  • / (index.html): Homepage pubblica. Pagina di benvenuto con link alle altre sezioni.
  • /login.html: Pagina di accesso per gli amministratori.
  • /calculator.html: Interfaccia del calcolatore classifiche. Questa pagina è protetta e accessibile solo agli amministratori autenticati.
  • /manual.html: Questa stessa pagina del manuale.
  • Altre pagine descrittive (es. `eventi.html`, `regolamento.html`) se presenti nel tuo progetto.

2. Controllo Accessi

Il sito implementa un sistema di autenticazione basato su JWT (JSON Web Tokens) per distinguere tra utenti pubblici e amministratori.

  • Utenti Pubblici: Possono visualizzare solo la homepage e altre pagine descrittive (es. regolamento, eventi).
  • Amministratori: Devono effettuare il login tramite /login.html. Una volta autenticati, possono accedere all'interfaccia del calcolatore classifiche (/calculator.html) e utilizzare tutte le funzionalità di elaborazione dati.

Processo di Login:

  1. Accedi a /login.html.
  2. Inserisci le credenziali di un utente amministratore.
  3. Se il login ha successo, verrai reindirizzato automaticamente a /calculator.html.
  4. Un cookie mwt_jwt verrà impostato nel tuo browser, identificandoti come amministratore per le successive richieste.
Nota sulla Sicurezza: Il cookie mwt_jwt è HttpOnly e Secure, il che significa che è accessibile solo via HTTP(S) e non via JavaScript, e viene inviato solo su connessioni HTTPS. Questo offre una protezione contro attacchi XSS.

Logout:

Sulla pagina /calculator.html è presente un pulsante "Logout". Cliccandolo, il cookie di sessione verrà cancellato e verrai reindirizzato alla pagina di login.

3. Utilizzo del Calcolatore Classifiche (Solo Amministratori)

L'interfaccia del calcolatore (/calculator.html) permette di elaborare i dati di una gara, aggiornare la classifica generale e generare report.

Workflow:

  1. Caricamento Gare: All'apertura della pagina, viene caricata la lista delle gare disponibili dal database D1.
    Seleziona una gara dal menu a tendina "Seleziona Gara:".
  2. Carica Dati: Clicca sul pulsante "Carica Dati". Questo recupererà tutti i dati grezzi (FIN, FAL, FTS) della gara selezionata dal database.
  3. Mappatura Segmenti: Una volta caricati i dati, apparirà un pannello "Mappatura Segmenti Gara". Qui dovrai classificare ogni segmento identificato nella gara come "Sprint ⚡", "KOM ⛰️" o "Nessuna". Questo è fondamentale per l'assegnazione dei punti Sprint e KOM.
    Importante: Questa mappatura è specifica per la singola elaborazione della gara. Le tue scelte non vengono salvate permanentemente per un riutilizzo futuro della stessa gara/tratta, ma vengono passate al backend per il calcolo attuale.
  4. Calcola Classifiche: Dopo aver classificato i segmenti, clicca sul pulsante "Calcola Classifiche". La logica di calcolo verrà eseguita sul backend (Cloudflare Function):
    • Verranno calcolate le classifiche della singola gara.
    • Verranno aggiornati i risultati cumulativi nel database D1, gestendo penalità per assenze e nuovi partecipanti.
    I risultati (sia della singola gara che i cumulativi aggiornati) verranno visualizzati immediatamente sotto il pannello di controllo.
  5. Download dei Risultati: Una volta calcolate le classifiche, i pulsanti di download diventeranno attivi:
    • Scarica Generale (JSON): Scarica il file cumulative_results.json aggiornato.
    • Scarica Gara (JSON): Scarica i risultati della singola gara in formato JSON.
    • Scarica Report (HTML): Genera e scarica un report HTML formattato, pronto per essere incorporato in altre pagine web.

4. Setup e Deployment (Per Sviluppatori/Proprietari)

Questa sezione riassume i passaggi tecnici necessari per configurare e mantenere l'applicazione. È rivolta all'amministratore del sito.

Configurazione Locale del Progetto:

  1. Clonare il repository GitHub contenente il progetto.
  2. Assicurarsi di avere Node.js e Wrangler installati globalmente.
  3. Nella root del progetto (mwt-cloudflare-app), eseguire npm install.

Database Cloudflare D1:

  1. Crea un database D1 dalla tua dashboard Cloudflare.
  2. Aggiorna il file wrangler.toml con il database_name e database_id del tuo database D1.
  3. Crea lo Schema: Esegui la migrazione SQL per creare le tabelle necessarie: 
    npx wrangler d1 execute <NOME_TUO_DB_D1> --file=./migrations/0001_initial_schema.sql
  4. Popola i Dati Iniziali:
    • Assicurati che i tuoi dati grezzi (cartella data/races e data/cumulative_results.json) siano presenti nella directory data/.
    • Genera lo script di popolazione SQL: 
      node populate_db.js
      Questo creerà il file populate_data.sql.
    • Esegui lo script SQL per popolare il DB: 
      npx wrangler d1 execute <NOME_TUO_DB_D1> --file=./populate_data.sql

Gestione Utenti (Amministratori):

  1. Crea un Utente Admin: Esegui lo script locale (ti chiederà username e password): 
    node functions/utils/create_admin_user.js
    Lo script ti fornirà un comando SQL INSERT.
  2. Inserisci l'Admin nel DB: Esegui il comando SQL generato contro il tuo database D1: 
    npx wrangler d1 execute <NOME_TUO_DB_D1> --command="<IL_COMANDO_SQL_GENERATO>"
  3. Configura JWT Secret:
    • Scegli una stringa lunga e casuale come chiave segreta per i JWT.
    • Nella tua dashboard Cloudflare, vai al progetto Pages -> Settings -> Environment variables.
    • Aggiungi una variabile con Nome: JWT_SECRET e incolla la tua chiave nel Valore. Assicurati che sia criptata e disponibile per gli ambienti Production e Preview.

Deployment su Cloudflare Pages:

Una volta configurato il progetto Cloudflare Pages e collegato al tuo repository GitHub, ogni volta che effettui un git push al branch configurato, Cloudflare Pages rileverà le modifiche e distribuirà automaticamente la nuova versione del sito.

5. Problemi Comuni e Risoluzione

  • 401 Unauthorized / 403 Forbidden: Assicurati di aver effettuato il login. Controlla che il JWT_SECRET sia impostato correttamente nelle variabili d'ambiente di Cloudflare Pages e che il tuo utente sia segnato come amministratore nel database D1.
  • Dati mancanti dopo il deploy: Verifica di aver eseguito correttamente i passaggi di "Deploy D1 Schema" e "Popola D1 con Dati".
  • Errori di Build: Controlla il log di build su Cloudflare Pages. Spesso sono dovuti a dipendenze mancanti (npm install non eseguito localmente prima del push) o a errori di sintassi nel codice.
