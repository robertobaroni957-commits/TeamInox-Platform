1️⃣ Preparazione del progetto

Assicurati di avere Git installato:

git --version


Vai nella cartella del progetto (dove ci sono riders.csv e riders.json):

cd /percorso/del/tuo/progetto


Controlla che il repository remoto sia configurato:

git remote -v


Se non vedi il tuo repository GitHub, collegalo:

git remote add origin https://github.com/username/nome-repo.git


Sostituisci username e nome-repo con quelli corretti.

2️⃣ Aggiungere i nuovi file

Sposta i nuovi file generati dallo script nella cartella corretta (ad esempio /data/):

mv riders.csv data/riders.csv
mv riders.json data/riders.json


Aggiungi i file a Git:

git add data/riders.csv data/riders.json


Controlla lo stato dei file aggiunti:

git status

3️⃣ Commit dei cambiamenti

Crea un commit con un messaggio chiaro:

git commit -m "Aggiornati riders.csv e riders.json"

4️⃣ Push su GitHub

Per aggiornare il ramo principale (main o master):

git push origin main


Sostituisci main con master se il tuo repository usa master.

Se vuoi forzare la sostituzione (sovrascrivendo eventuali modifiche remote):

git push origin main --force


⚠️ Attenzione: --force cancella la cronologia remota di quel ramo.

5️⃣ Verifica

Apri il repository GitHub nel browser.

Vai nella cartella /data/ e controlla che i file riders.csv e riders.json siano aggiornati con i nuovi dati.

💡 Suggerimento pratico:
Puoi automatizzare questa procedura aggiungendo uno script bash come:

#!/bin/bash
# Sposta i file aggiornati
mv riders.csv data/riders.csv
mv riders.json data/riders.json

# Aggiunge e committa i file
git add data/riders.csv data/riders.json
git commit -m "Aggiornati riders.csv e riders.json"

# Push su GitHub
git push origin main


Poi ogni volta che rigeneri i file basta eseguire lo script.
